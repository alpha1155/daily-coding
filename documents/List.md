### 一、Java并发篇（必问Top1，几乎每面必炸）

1. volatile 和 synchronized 的区别？ → volatile只保证可见性和有序性，不保证原子性；synchronized既保证可见性+有序性，也保证原子性。volatile修饰变量，synchronized可修饰代码块/方法。 高阶：volatile底层靠内存屏障阻止指令重排 + Lock前缀指令刷新缓存；synchronized靠Monitor对象。
2. 什么是指令重排？原因？会造成什么问题？ → CPU和编译器为了优化执行效率进行的指令乱序。原因：as-if-serial（单线程语义不变）+ 处理器乱序执行。 典型问题：DCL单例失败（instance = new Singleton() 不是原子操作，可能看到半初始化的对象）
3. 偏向锁、轻量级锁、重量级锁的区别和升级过程？ → 无锁 → 偏向锁（无竞争时偏向第一个线程）→ 轻量级锁（CAS自旋抢锁）→ 重量级锁（膨胀为Monitor，线程阻塞） 高阶：MarkWord结构变化 + 锁升级是不可逆的（JDK15后偏向锁可撤销）
4. 谈谈你对AQS的理解？AQS如何实现可重入锁？ → AQS是AbstractQueuedSynchronizer，所有阻塞锁的基石。核心：一个volatile int state + CLH队列。 可重入：ReentrantLock通过state累加 + 当前线程判断owner是否是自己。
5. A、B、C三个线程如何保证同时执行、依次执行、交错执行？
   - 同时执行：CountDownLatch count=3，三个线程start后await(0)
   - 依次执行：T2.join()放T1里，T3.join()放T2里 或 单线程线程池
   - 交错执行：CyclicBarrier parties=3 或 两个Semaphore(0)

### 二、JVM篇（必问Top2，阿里P6+必深挖）

1. 双亲委派机制及作用？如何打破？ → ClassLoader加载类时先委托父加载器，防止重复加载 + 防止核心类被篡改。 打破方式：自定义ClassLoader重写loadClass() 或 SPI（线程上下文类加载器）
2. 对象从创建到死亡的全过程？ → new → TLAB分配（逃逸则Eden）→ Minor GC存活进S区 → 15次GC后进老年代 → Full GC回收
3. 如何判断对象是垃圾？GC Roots有哪些？ → 可达性分析算法。GC Roots：虚拟机栈本地变量、方法区静态变量、常量引用、本地方法栈JNI引用、已启动未停止的线程
4. 垃圾回收算法 + 回收器 + 三色标记漏标错标解决方案
   - 算法：标记-清除、标记-整理、复制
   - 回收器：Serial、ParNew、CMS（标记-清除+增量并发）、G1（Region+Remember Set）、ZGC（染色指针）
   - 三色标记漏标解决方案：原始快照（SATB）或 增量更新（G1）
5. JVM调优常用参数？ → -Xms = -Xmx（避免堆伸缩）、-Xmn（新生代）、-XX:+UseG1GC、-XX:MaxGCPauseMillis=200、-XX:+PrintGCDetails、-XX:+HeapDumpOnOutOfMemoryError

### 三、Redis篇（2025年最火中间件）

1. Redis为什么这么快？到底是单线程还是多线程？ → 6.0前是单线程（Reactor模型+IO多路复用），7.0后IO线程多线程了，主线程依然单线程处理命令。 快的原因：纯内存 + 单线程无锁 + IO多路复用 + 高效数据结构
2. Redis持久化策略？AOF有了还要RDB吗？ → RDB（快照，fork子进程）适合冷备、AOF（命令追加）可恢复更多数据。 生产必开混合持久化（4.0+）：重启时加载RDB + 增量AOF
3. 缓存穿透、击穿、雪崩 + 终极解决方案
   - 穿透：布隆过滤器 + 空值缓存
   - 击穿：热点本地缓存（如Caffeine）+ 互斥锁重建
   - 雪崩：随机过期 + 二级缓存 + 限流熔断
4. 缓存与数据库双写一致性终极方案（2025最新答案） → 延迟双删 + Canal订阅binlog异步删缓存（最优） 或读修旁路：读到空值时加载到缓存并返回
5. Redisson分布式锁原理 + 为什么用Lua脚本？ → setnx + expire（不原子）→ 改用Lua脚本保证原子性 + 可重入（hash结构）+ 看门狗自动续期

### 四、消息队列MQ篇（P7+必问）

1. Kafka为什么这么快？（送分也是送命题） → 顺序写磁盘 + 零拷贝（mmap/sendfile）+ PageCache + 批量压缩 + 分区并行
2. MQ如何保证100%不丢消息？ → 生产：同步确认（ack= -1）+ 重试 + 落库异步刷盘 → 消费：消费成功后再提交offset（手动ack）
3. MQ如何保证幂等性 + 顺序性
   - 幂等：全局唯一消息ID + Redis去重表/数据库唯一约束
   - 顺序：同一业务单分区 + 单线程消费（Kafka/RocketMQ都支持）
4. 分布式事务最终一致性方案 → RocketMQ事务消息（半消息 + 二阶段提交） → Seata AT（最透明，2025主流）

### 五、MySQL篇

1. 聚簇索引 vs 非聚簇索引？为什么InnoDB必须有主键？ → 聚簇：叶子节点存整行数据（InnoDB主键索引） → 非聚簇：叶子节点存主键值（MyISAM、辅助索引） → 没有主键会生成6字节RowID，浪费空间且影响性能
